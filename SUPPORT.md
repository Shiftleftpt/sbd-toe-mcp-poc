# Suporte

## Canais recomendados

| Tema | Canal |
| --- | --- |
| bug técnico ou falha de runtime | abrir issue em [Issues -> Bug report](../../issues/new?template=bug_report.yml) |
| melhoria de documentação ou setup | abrir issue em [Issues -> Documentation](../../issues/new?template=documentation.yml) |
| proposta de funcionalidade ou melhoria | abrir issue em [Issues -> Enhancement](../../issues/new?template=enhancement.yml) |
| dúvidas de uso e integração | GitHub Discussions quando ativado; até lá, usar a template de documentação |
| vulnerabilidades | `security@shiftleft.pt` |

## Notas

- Este repositório é distribuído principalmente via GitHub Releases.
- O bundle inclui `dist/` e `data/publish/`; reportes de suporte devem indicar se o problema foi observado num release asset ou num checkout de source.
- Para questões de conteúdo editorial do manual, identifica sempre a pergunta concreta e, se possível, o record ou capítulo relacionado.

## SLA indicativo

- issues e PRs: até 5 dias úteis
- reportes de segurança: até 72 horas
